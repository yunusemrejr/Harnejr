## Visual direction

Harnejr should feel like a serious local AI harness, closer to a professional developer tool than a demo chat app.

Rules:

- gray and cream tones only;
- no gradients;
- no glows;
- no blinking indicators;
- no emoji icons;
- no decorative fake status labels;
- controls should be separated by runtime purpose.

The UI may borrow professional product-shell patterns from mature AI coding tools: strong sidebar, clear composer, dense configuration panels, calm surfaces, useful proof cards, and visible runtime state. It should not copy another product wholesale.
